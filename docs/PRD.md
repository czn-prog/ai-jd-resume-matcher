# PRD - AI简历JD匹配助手
## 1. 用户画像
1）C端求职者：校招/转行人群，不清楚岗位核心要求，简历缺乏针对性
2）运营/HR后台：批量筛选简历，需要量化匹配标准
## 2. 核心需求
1）单条JD智能解析，结构化输出岗位能力要求
2）单简历匹配打分，输出优化方案与面试题
3）批量简历AI评测，导出标准化数据报表
## 3. 交互流程
1. 用户粘贴JD → 点击解析，生成结构化岗位信息
2. 粘贴简历，一键生成匹配报告
3. 后台上传简历数据集，批量评测导出结果
## 4. 非功能需求
1. 数据隐私：本地运行，不上传简历至第三方服务器
2. 异常兜底：空白输入、超长文本给出友好提示
3. 易用性：分步操作，降低学习成本
## 5. 风险与解决方案
风险1：大模型输出不稳定 → 批量评测机制持续优化Prompt
风险2：简历信息泄露 → 纯本地运行，无云端存储
