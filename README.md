# AI简历JD匹配助手 | AI产品经理求职Demo
## 项目简介
面向求职人群的AI工具，完整复刻AI产品经理全工作链路：用户调研、PRD设计、交互打磨、AI评测方案、上线迭代，贴合AI产品岗招聘JD全部职责。
解决痛点：求职者投递岗位时，无法精准匹配JD要求，简历缺少核心关键词，面试无针对性准备；HR批量筛选简历效率低。

## 技术栈
Python + Gradio 前端交互 + LangChain + LLM API
无需GPU，本地一键运行。

## 本地部署步骤
### 1. 创建并激活虚拟环境
```cmd
python -m venv venv
venv\Scripts\activate
```
### 2.安装项目依赖
```cmd
pip install -r requirements.txt
```
### 3.配置大模型密钥
在`.env`文件中填入你的 LLM API Key、接口地址、模型名称
### 4. 启动项目
```cmd
python main.py
```

## 产品功能模块

### 1. C 端求职者模块

- JD 自动结构化拆解：提取硬性门槛、业务能力、AI 技能、加分项
- 简历智能匹配打分：输出综合匹配度、个人优势、缺失关键词
- 自动生成简历优化建议 + 岗位专属模拟面试题

### 2. 后台 AI 评测模块

- 支持批量导入多份简历数据集
- LLM 自动量化三项指标：综合匹配分、缺失能力数量、AI 经历匹配度
- 一键导出 CSV 评测报表，用于分析模型效果、沉淀 Bad Case

## 产品设计思路

1. 需求梳理：区分求职者、HR 两类用户，完成用户调研并输出完整 PRD，搭建可交互原型；
2. 体验优化：页面分层设计，分步操作引导，针对空输入、超长文本做异常兜底交互；
3. AI 策略评测：独立设计完整评测体系，批量实验定位 Prompt 缺陷，形成数据迭代闭环；
4. 项目全流程模拟：覆盖需求评审、交互开发、模型效果测试、灰度上线、线上反馈复盘；
5. 长期创新规划：结合 Agent、多模态大模型规划迭代版本，平衡落地成本与业务价值。

## 项目落地风险与解决方案

1. 大模型输出不稳定：搭建批量评测能力，持续迭代 Prompt 优化打分一致性；
2. 用户简历隐私泄露：纯本地运行，简历数据不对外上传存储；
3. 长文本解析截断：增加文本分段预处理逻辑，保证 JD、长简历完整识别。

## 迭代规划

V1 现有版本：JD 解析、单简历匹配、批量 AI 评测导出报表;
V2 体验升级：支持 PDF/TXT 简历上传、会话历史记录、完善异常交互;
V3 AI 创新拓展：Agent 自动改写简历、简历敏感信息脱敏、多模型横向对比评测.
